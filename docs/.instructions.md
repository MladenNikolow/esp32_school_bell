# ESP32 School Bell Firmware Context

When working on any source file in this project, be aware of the system architecture:

## Key Architecture

- **Platform**: ESP-IDF v5.4.0, FreeRTOS, ESP32-S3 (16MB Flash, 8MB PSRAM)
- **Board**: Waveshare ESP32-S3-Touch-LCD-4 (480×480 IPS, capacitive touch)
- **Pattern**: Opaque handles (`typedef struct _X_RSC_T* X_H`) throughout
- **Boot**: `app_main()` → `AppTask_Create()` → 4-phase init → event loop

## Component Map

| Component | Purpose | Key API |
|-----------|---------|---------|
| **AppTask** | Orchestrator, 4-phase boot | `AppTask_Create()` |
| **FileSystem** | FatFS `/react/` + SPIFFS `/storage/` | `FatFS_Init()`, `SPIFFS_ReadFile/WriteFile()` |
| **Generic** | Error codes | `APP_SUCCESS`, `APP_ERROR_*` |
| **NVS** | Key-value persistence | `NVS_Open/Read/Write/Commit/Close()` |
| **RingBell** | GPIO bell + panic mode | `RingBell_Init/Run/Stop/RunForDuration/SetPanic()` |
| **Scheduler** | Bell scheduling, shifts/holidays/exceptions | `Scheduler_Init/ReloadSchedule/GetStatus()` |
| **TimeSync** | NTP sync + timezone | `TimeSync_Init/IsSynced/SetTimezone()` |
| **TouchScreen** | LVGL UI: screens, services, i18n | `TouchScreen_Init/ShowDashboard()` |
| **WebServer** | HTTP server + REST API + auth | `Ws_Init()`, `auth_require_session()` |
| **WiFi_Manager** | WiFi credentials (NVS) | `WiFi_Manager_Init/SaveCredentials()` |

## Auth System

- Session-based: HttpOnly cookies, 3 max sessions, 24h expiry
- CSRF: `Content-Type: application/json` + `X-Requested-With: XMLHttpRequest` on POST/PUT/DELETE
- Guard pattern: `auth_require_session()` → `auth_csrf_check()` per handler
- Credentials: Kconfig `WS_AUTH_USERNAME` / `WS_AUTH_PASSWORD`

## Patterns to Follow

- Use opaque handles — never access `_RSC_T` internals from outside a component
- Use `APP_SUCCESS`/`APP_ERROR_*` return codes for custom component APIs
- Use `esp_err_t` for ESP-IDF wrapper APIs
- Use cJSON for JSON serialization (caller must `cJSON_Delete()` returned objects)
- Use NVS namespace isolation per subsystem
- SPIFFS for user data at `/storage/`, FatFS for web assets at `/react/`
- TouchScreen Services as bridge layer — UI never calls backend components directly
- All web API handlers must call `auth_set_security_headers()` first
